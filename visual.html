Processing

What is Processing?

Processing is a piece of software written for artists, by artists. It allows the user to create 2d and 3d images, videos, art installations, 3d prints, robots... the list goes on! It uses the Java language and a large library of functions to remove the painful processes of asking a computer to draw to pixels on the screen and lets the user simply code away! The software is free and is available from the Processing web site.

There is a lot of potential in this software, I have been scratching the surface of possibilities. As I create more work worth sharing I'll add new sections below. My most recent area of interest has been playing with the idea that the numbers used to store digital pictures (i.e. the RGB components of each pixel) could be used in novel ways to either manipulate the image or create brand new ones.

Colour Shifting

I've been experimenting with processing images by moving each pixel by an amount determined by the amount of red, green or blue in that pixel. Below are some of the more interesting examples.

Twisted Face. This has been shifted to the left depending on how much red is in each pixel. I like the appearance of the 2d image being distorted in 3 dimensions (though this isn't what is actually happening).

In the top photo each pixel is separated into it's component colours and shifted based on the brightness of each. The lower photo has been shifted based on a single component of each pixel.

Pixel Lines

This has been my favourite experiment to date. In this I am playing with drawing lines from individual pixels to another point. Below are some of the more interesting variations.

Quad Portrait. The source photo is that of a girl with a shaven head in profile in front of a brick wall. The image above is a still from the end of the process that can take around half an hour to form. A pixel is chosen from the source photo and a translucent line of that colour is drawn to the corner, this is done 4 times, one for each corner forming 4 separate images. The pixel being selected meanders around the photo leaving a trail of lines and eventually exposing the image. I really like how it defines different parts of the image in each quarter of the final piece, a ghostly face appearing from a bundle of straight lines.

Explosion (top). Purple Landscape (bottom). These pieces were made by analysing a photo by choosing a pixel at random then finding up to 100 other pixels in the source image with the exact same colour. It then connects the 100 pixels to the first one with a line of the common colour. The idea was to create connections between regions of the pictures and ultimately an interesting abstraction of the source photo.

Point Tracer

The idea behind these images was to use the information already in photos to generate patterns. Each line of pixels in a photo is treated as a series of instructions whereby a line of each rgb colour component are drawn from a single point of origin. The amount of each colour component in the line is used to steer the line left or right. This is a long process with each line being drawn individually, eventually making the images below. The images are all variations on the theme with differing source pictures, lines thickness, run times, blend modes etc. What I hoped to find was that because each line of pixels is often very similar to the line above it, the images would have some kind of consistency and interesting patterns would form throughout the image.
